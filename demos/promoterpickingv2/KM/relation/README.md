# KM Peak-Gene 一对一关系

## 文件说明

### 主要文件

- **`one_to_one_relations.csv`**: 合并后的一对一关系文件（推荐使用）
  - 包含所有4个样本（C1, C3, O2, O3）的合并结果
  - 以基因为主题，按C1、C3、O2、O3优先级选择
  - 每个基因只出现一次（真正的一对一关系）
  - 包含 `source_sample` 列，标记每个关系的来源样本

- **`one_to_one_summary.txt`**: 合并后的统计摘要

### 原始文件（按样本分别）

- `one_to_one_relations_C1.csv` - C1样本的一对一关系
- `one_to_one_relations_C3.csv` - C3样本的一对一关系
- `one_to_one_relations_O2.csv` - O2样本的一对一关系
- `one_to_one_relations_O3.csv` - O3样本的一对一关系

## 合并策略

1. **优先级顺序**: C1 > C3 > O2 > O3
2. **选择逻辑**: 对于每个基因，按优先级顺序查找，找到第一个匹配的关系就停止
3. **结果保证**: 每个基因只出现一次，确保真正的一对一关系

## 统计结果

- **总关系数**: 1790
- **唯一基因数**: 1790
- **唯一peak数**: 1474

## 使用建议

**推荐使用 `one_to_one_relations.csv`**，因为：
- 已经合并了所有样本，不需要分别处理4个文件
- 每个基因只出现一次，确保一对一关系
- 包含来源样本信息，便于追溯

## 重新生成

如果需要重新生成合并文件，运行：
```bash
python merge_one_to_one_by_gene.py
```
