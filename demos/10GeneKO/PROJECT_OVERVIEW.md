# GeneKO 项目总览

## 🎯 项目简介

GeneKO是一个专为酵母基因敲除（Gene Knockout）数据分析设计的系统，能够将原始实验条件转换为机器学习模型可用的标准化特征，并生成与训练数据格式完全一致的numpy文件。

## 🏗️ 系统架构

### 核心组件
```
┌─────────────────┐    ┌─────────────────┐    ┌─────────────────┐
│   数据预处理     │ →  │   特征编码      │ →  │   numpy文件     │
│ process_geneko_ │    │ build_geneko_   │    │   输出         │
│ data.py         │    │ numpy.py        │    │                │
└─────────────────┘    └─────────────────┘    └─────────────────┘
         ↓                       ↓                       ↓
   标准化数据格式            60维特征向量           模型输入格式
```

### 数据流程
1. **输入**: 原始Excel/CSV数据（实验条件 + 基因表达）
2. **预处理**: 标准化数值、处理缺失值、验证数据一致性
3. **编码**: 数值标准化 + 分类独热编码
4. **输出**: 标准化的numpy文件 + 完整的编码信息

## 📊 特征系统

### 特征维度
- **总特征数**: 60
- **数值特征**: 7个（时间、温度、浓度等）
- **分类特征**: 53个（培养基、碳源、氮源、药物等）

### 编码方式
- **数值特征**: StandardScaler标准化
- **分类特征**: OneHotEncoder独热编码
- **表达值**: log1p转换，处理两种类型的0值

### 两种类型的0值处理
1. **NaN转0**: 缺失值用0填充，确保数据完整性
2. **负值转0**: 生物学上不合理的负表达值强制设为0

## 🚀 使用方式

### 快速开始
```bash
cd GeneKO
python run_geneko_pipeline.py  # 一键运行
```

### 分步执行
```bash
python process_geneko_data.py      # 数据预处理

```

## 📈 输出格式

### numpy文件结构
```
形状: (samples, peaks, features)
- samples: 样本数量
- peaks: ATAC-seq peak数量  
- features: 284(ATAC) + 60(条件) + 1(表达) = 345
```

### 特征组合
- **ATAC特征**: 284维，来自ATAC-seq数据
- **条件特征**: 60维，编码后的实验条件
- **表达值**: 1维，log转换后的基因表达值
