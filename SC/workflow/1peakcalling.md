SC/1peakcalling workflow 描述

这一步的输入是已经完成比对和整理的 reads 文件，通常是 BED 格式。可以理解为：文件里至少要有 reads 的基因组位置，也就是染色体、起始位置、终止位置这些基本信息；如果已经是整理好的 peak 相关文件，也可以作为这一步的输入来源。

这一步使用 `MACS2 callpeak` 来做 peak calling，也就是从 reads 中识别富集区域。

常见参数包括：
`-f BED` 用来指定输入格式；
`-g` 用来指定有效基因组大小；
`-q` 用来控制 peak 的筛选阈值；
`--nomodel --shift --extsize` 用来手动设置片段位置和长度；
`--keep-dup` 用来控制重复 reads 的处理方式；
`--bw` 用来设置背景平滑相关参数。

这里的 `Q` 值通常会根据数据特性来调整。对于酵母这种基因组较小、信号较密集的数据，阈值往往会设得比较宽松一些，这样可以尽量保留更多潜在 peak，方便后续把峰划分得更细，也更符合下游分析需求。

这一步的输出一般包括三类文件：
`*_peaks.xls`，保存 peak 的详细统计信息；
`*_peaks.narrowPeak`，标准的 peak 区间文件；
`*_summits.bed`，保存每个 peak 的 summit 位置。

这三类文件的含义分别是：
`xls` 主要用于查看每个 peak 的统计信息，比如位置、峰顶、富集程度和显著性；
`narrowPeak` 是标准区间结果，后续最常用，适合做注释、可视化和区间交集；
`summits.bed` 只保留每个 peak 的 summit，也就是信号最强的位置，适合更精细的后续分析。

整体上，这一步的作用就是把比对后的 reads 转成可用的 peak 文件，为后续 motif 分析、区域注释和功能分析做准备。
