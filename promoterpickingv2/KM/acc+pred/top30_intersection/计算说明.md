# 711的计算过程

## 计算步骤

### 步骤1: 一对一关系
- **输入**: 4个sample的一对一关系文件（C1, C3, O2, O3）
- **处理**: 按C1、C3、O2、O3优先级合并，每个基因只保留第一个找到的关系
- **结果**: 
  - 1790个基因
  - 1474个唯一peak
  - 1790条关系（每个基因一条）

### 步骤2: 准确度计算
- **输入**: 4个sample的预测数据（predictions_matrix_{C1,C3,O2,O3}.csv.csv）
- **处理**: 从预测数据中提取这1474个peaks的预测值和真实值，计算pearson和MAE
- **结果**: 
  - 1474个peaks的准确度指标
  - 分正负链分别计算

### 步骤3: Top30%选择
- **输入**: 1474个peaks的准确度指标
- **处理**: 从1474个peaks中选top30%（按正负链分别选择）
  - 正链：选pearson_pos最高的30%
  - 负链：选pearson_neg最高的30%
- **结果**: 
  - 886条记录（443条正链 + 443条负链）
  - 751个唯一peak

### 步骤4: 交集计算
- **输入**: 
  - 一对一关系：1790条（1790个基因，1474个唯一peak）
  - Top30%准确率：886条（751个唯一peak）
- **处理**: 按(peak_id, strand)匹配，求交集
- **结果**: 
  - **711条关系**（711个基因，654个唯一peak）

## 关键点

1. **不是从4个sample的所有peaks中选top30%**
   - 而是从**一对一关系中的1474个peaks**中选top30%
   - 这1474个peaks已经对应了1790个基因

2. **Top30%选择是按正负链分别选的**
   - 从1474个peaks中，正链选top30%（443条）
   - 从1474个peaks中，负链选top30%（443条）
   - 总共886条记录，751个唯一peak

3. **为什么交集是711而不是886？**
   - Top30%有886条（按正负链分别选）
   - 但一对一关系中，某个peak可能只有正链的基因（或只有负链的基因）
   - 所以Top30%中某个peak的某个strand可能在一对一关系中没有对应的基因
   - **886条Top30% → 711条交集（减少了175条）**

4. **匹配键是(peak_id, strand)**
   - 同一个peak的正链和负链是分开匹配的
   - 711条关系 = 711个(peak_id, strand)组合
   - 654个唯一peak（有些peak在正负链都有匹配）

## 总结

**711 = 一对一关系(1790条) 与 Top30%准确率(886条) 的交集**

- 一对一关系：1790个基因，1474个唯一peak，1790条关系
- Top30%准确率：从1474个peaks中选top30%（按正负链分别）→ 886条记录，751个唯一peak
- 交集：按(peak_id, strand)匹配 → **711条关系，711个基因，654个唯一peak**

**为什么交集会减少？**
- Top30%选择时，按正负链分别选（正链top30%，负链top30%）
- 但一对一关系中，某个peak可能只有正链的基因（或只有负链的基因）
- 所以Top30%中某个peak的某个strand可能在一对一关系中没有对应的基因
- 因此：886条Top30% → 711条交集（减少了175条）
